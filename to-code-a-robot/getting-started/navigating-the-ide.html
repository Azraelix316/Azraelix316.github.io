Tip

Customize!!!
Obviously, go ahead and customize your IDE setup - WPIlib VScode is VScode but with additional features, so you can do all the VScode things like download plugins and change your editor to light mode (if you woke up and chose violence).
You'll be looking at this screen for a long, long time so do whatever you need to be happy with it (no judgement (except if you use light mode, everyone except for Jard will judge you))

Command Palette

Now that you've finished downloading WPIlib, you should learn how to create projects and use the command palette. You can access all commands using the shortcut Ctrl + Shift + P, but if you only want to see WPIlib based commands you can click the WPIlib logo

This brings up a list of commands - the most important ones are build robot code and deploy robot code. Building the code checks it for errors, and deploying the code deploys it to the robot.

In the sidebar, there's also a WPIlib logo, and when you click it it shows the vendor dependencies. Vendor dependencies (vendordeps) are used to add additional function on top of default WPIlib function, such as CTRE Phoenix's libraries, which add motion magic (motion profiles).

Creating your first project

Go into the command palette and create a new project. When you get to the project creation screen, select the template option, then java, then "Command Robot Skeleton (Advanced)". This gives you a virtually blank slate to start from.
Follow the rest of the directions (team number for R1 is 8503) to make the project!
Now, you're ready to move into Git and Github and start doing actual hands-on work.
